== The wiki — three tracks

=== 👤 Users (support)  — plain-language, task-first
* Getting started: the *Core* on a spare device ▢
* Right-size to your line (2.5GbE only for gigabit+) ▢
* Troubleshooting + the one-button *diagnostic-bundle* export ▢
* *"Your site's been hijacked — recover, in order"* (incident response) ▢
* Reputation hygiene: Safe Browsing / SmartScreen; staying off blocklists ▢

=== 🕸️ IndieWebbers (the special extra)
* Make your page with *Ddraig* (accessible, WCAG-AAA-capable, no Node) ▢
* Publish to Cloudflare Pages — free, likely green-hosted ▢
* Your own domain + mail-DNS done right (SPF/DKIM/DMARC via the module) ▢
* `security.txt`, sitemap, Search Console — being findable *and* trusted ▢

=== 🛠️ Developers / platform maintainers (incl. Frontier)
* The two-box estate, release channels (`main`/`alpha`), and *modules* ▢
* Frontier: SPA/SDP, the ODoH pool, the bastion, full observability ▢
* *Bustfiles* + `network-ambulance` — recovery-as-code ▢
* Making the box legible to your own LLM (the diagnostic bundle, `0-AI-MANIFEST`) ▢
* Contributing · governance · the pin/bump flow · OpenSSF ▢

== Fill order (suggested)
Users track first (getting-started + the IR page), then IndieWeb (Ddraig + publish),
then maintainer (Frontier modules). One page at a time, reviewed as we go.
